HYBRID storage driver (compute)

A HYBRID driver tells Dataloop which compute and which named storage on that compute (for example NFS or S3) should back datasets in this project. After you create it, you reference the driver when creating a dataset and in the stress-test workflow (for example via driverId / hybrid storage settings).

What you need:

• Driver name — a unique name for this driver in the project (required).

• Compute storage — the list comes from this service’s resolved compute and its storage-driver (or dataset) plugin, same idea as in the platform compute UI. One storage → used automatically; several → choose in the dropdown (names may show a type such as NFS or S3).

• Path on storage — optional subfolder on that storage. If you leave it blank, the service applies its default folder for this flow.

Check existing HYBRID drivers

Use Refresh to load HYBRID drivers for this project. By default the list is limited to drivers whose compute matches this service’s compute (same resolution as loading storages above), so you only see drivers that belong to this stress-test setup.

Optional environment overrides (service / deployment): HYBRID_DRIVERS_PROJECT_WIDE=1 lists every HYBRID driver in the project (all computes). HYBRID_DRIVERS_ALLOW_SDK_WITHOUT_PROJECT_ID=1 can include drivers that have no project id in the API if their id appears in the SDK list. Use these only when you understand the broader listing.

Create HYBRID driver

Submits a create request through this panel’s API. The service resolves the compute (from your environment / service) and creates the HYBRID driver with the name, storage, and path you enter below.
